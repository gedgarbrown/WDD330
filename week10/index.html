Client side Form Validation

What is form validation?
Validation of any user entered data to ensure it is correct.
Can be done either client side or server side
Validation is done to ensure the right data is gathered
To protect user's data
And to protect ourselves

Different types of client-side validation
Built in form validation -- uses features from HTML5
Javascript -- uses functions from js to validate

Using Built-in form validation
validation attributes:
required
minlength / maxlenght
min / max
type
pattern - follows a regular expression
CSS psuedo-class -- :valid
CSS psuedo-class -- :invalid or other classes :out-of-range
only valid can be submitted, browser will block submission of invalid

Validation using Javascript

The constraint Validation API
  • HTMLButtonElement (represents a 'button' element)
  • HTMLFieldSetElement (represents a 'fieldset' element)
  • HTMLInputElement (represents an 'input' element)
  • HTMLOutputElement (represents an 'output' element)
  • HTMLSelectElement (represents a 'select' element)
  • HTMLTextAreaElement (represents a 'textarea' element)

Allows the following properties: validationMessage -- returns localized message
validity -- returns Validity state

----Validity States:
----patternMismatch
----tooLong
----tooShort
----rangeOverflow
----rangeUnderflow
----typeMismatch
----valid
----valueMissing

Methods:
checkValidity() - returns true if there are no problems
setCustomValidity(message) -- add custom error message
